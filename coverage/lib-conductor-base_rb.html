 66     # * Everything happens in a transaction so it can all be rolled back

 71     # * If we are dealing with a new record, we must save it first (without
 72     #   validation or callbacks) in order to get an id from the database, which
 73     #   allows through associations to save properly
 74     # * If there is a failure we need to switch the resource back to thinking it's
 75     #   a new record, because the transaction rollback will ensure there is no
